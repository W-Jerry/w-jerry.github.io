记录一次HyperFlex的安装经历

发表于 2017-09-13

在这次的HyperFlex的安装过程中，碰到了不少问题，也让我学到了不少。

首先你需要先完成这些步骤：

登录hxinstaller之后，输入UCS Manager Host Name和vCenter Server就碰到了第1个问题.

问题1：发现这2个FQDN都无法解析，使用SSH登录故hxinstaller之后，发现DNS配置有误，修改之后可以正常解析。这里有个小tips，加一个DNS Server记录就可以了，后面在配置的时候也是用这个，就一定不会出错。当配置3个DNS的时候，后面会提示找不到DNS。

安装的过程主要分为一下几个步骤：Start->Validations->UCSM Configuration->Hypervisor Configuration->Deploy Validation->Deploy->Create Validation->Cluster Creation

在Validations的过程中，碰到过一次问题是由于DNS配置了3个DNS Server记录造成无法找到DNS造成的

USCM Configuration主要是完成FI里面的网络VLAN和server profile的一些配置工作。

Hypervisor Configuration主要是完成Host的ESXI的安装和配置工作。

Deploy Validation和Deploy主要是完成Storage VM的安装和配置，这里使用的Ansible的自动化来完成部署。

上面的步骤是Cisco HX Data Platform Installer和FI和HyperFlex进行通信完成。

Create Validation的过程中，碰到的一次完成是由于没有清楚Storage VM Cluster造成的，检测发现VIP已经存在。

Cluster Creation这里开始是vCenter开始和Host还是有Storage VM开始通信完成配置。这里碰到了第2个问题.

问题2：出现404错误，通过分析网络包，发现这里是原理是Java开始调用vCenter的8089的SDK的端口，然而由于这个端口被其他应用程序所占用，vCenter在安装的时候，自动+1使用8090端口，可以通过vCenter的eam日志看到。在这里一直试图想修改端口到8089，或者Java端口修改到8090来解决，结果都没有结果。最后的解决方案在vCenter上vpxd-rhttpproxy-endpoint.conf的配置文件里面添加一行sdkTunnel:8089 localtunnel 8090 allow reject来实现Http Proxy的redirect来实现的。

然后就碰到了第3个问题。

问题3：Java exception，通过分析网络包，发现在80的port上会出现一些rst的标志，开始怀疑是vCenter有问题，然后果然发现vCenter的一些服务的异常，这里比较难以修复，所以最快的解决方案，重新安装了一台vCenter，然后把HyperFlex cleanup之后，重新安装，成功完成。

总结：虽然这里都是几个小问题，但是导致安装的过程中碰到了不少问题。

1.确保防火墙打开。

2.确保vCenter全部功能正常。

3.确保hxinstaller安装配置正确，尤其是DNS的配置。

4.检查vCenter的SDK的8089端口功能正常，如果被占用，查看现在的端口情况，并且修改rhttpproxy的配置文件，使其能实现http的redirect。
